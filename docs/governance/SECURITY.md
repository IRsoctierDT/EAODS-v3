# Security Policy

## Reporting

Do not disclose vulnerabilities through public issues. Use GitHub's private security reporting capability where available.

## Prohibited repository content

Never commit:

- passwords, tokens, private keys, or certificates;
- production configuration containing secrets;
- customer or regulated data;
- sensitive incident evidence;
- unredacted infrastructure details that create material risk.

## Secure contribution baseline

Treat external content as untrusted, validate generated documentation, use least privilege, and require human approval for production-impacting recommendations.

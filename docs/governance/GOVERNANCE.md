# EAODS Governance

## Authority

The repository owner retains final publication authority. Material architecture, security, governance, and commercialization changes require documented human review.

## Change classification

- **Patch:** Editorial correction or clarification.
- **Minor:** Additive framework or architecture enhancement.
- **Major:** Breaking governance, taxonomy, constitutional, or architecture change.

## Required document controls

Every normative framework artifact shall include:

1. YAML front matter.
2. Purpose and scope.
3. Enterprise workflow.
4. Integration points.
5. QA checks.
6. Human review gate.

## Decision records

Significant design decisions shall be documented under `architecture/adr/`.
